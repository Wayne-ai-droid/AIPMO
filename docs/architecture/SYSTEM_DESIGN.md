## 1. 系统概述

AIPMO是一个AI驱动的项目管理办公室系统，通过多Agent协作完成产品需求、设计、开发和测试。

### 1.1 核心组件

```
┌─────────────────────────────────────────────────────────┐
│                    AIPMO System                         │
├─────────────────────────────────────────────────────────┤
│  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐   │
│  │PM Agent │  │Arch     │  │Design   │  │Dev      │   │
│  │         │  │Agent    │  │Agent    │  │Agent    │   │
│  └────┬────┘  └────┬────┘  └────┬────┘  └────┬────┘   │
│       │            │            │            │         │

│                      │                                  │
│              ┌───────┴───────┐                          │
│              │  GitHub API   │                          │
│              │  Repository   │                          │
│              └───────────────┘                          │
└─────────────────────────────────────────────────────────┘
```

## 2. 技术架构

### 2.1 数据流

```
User (GitHub Issue)
    ↓
PM Agent (分析/分配)
    ↓
Arch/Design/Dev/QA (执行)
    ↓
GitHub (代码/文档提交)
    ↓
PM Agent (验收/关闭)
```

### 2.2 存储结构

```
docs/
├── prd/           # 产品需求文档
├── architecture/  # 架构设计文档
└── design/        # UI/UX设计文档

src/               # 源代码
tests/             # 测试用例
```

## 3. Agent职责

| Agent | 职责 | 输出 |
|-------|------|------|
| PM | 需求分析、任务分配、进度跟踪 | PRD、任务清单 |
| Arch | 系统架构、技术选型 | 架构设计文档 |
| Design | UI/UX设计、交互规范 | 设计稿、规范文档 |
| Dev | 代码开发、单元测试 | 源代码 |
| QA | 测试用例、质量验证 | 测试报告 |

## 4. 协作流程

1. **Issue创建** → 用户提交需求
2. **PM分析** → 需求分解、任务分配
3. **并行执行** → 各Agent独立工作
4. **代码审查** → PR合并到main分支
5. **QA验证** → 测试验收
6. **PM关闭** → Issue完成
